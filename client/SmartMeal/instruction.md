# 📱 SmartMeal — Android Client

Этот раздел репозитория содержит исходный код нативного Android-приложения «Умный ужин». Клиентская часть отвечает за взаимодействие с пользователем: от планирования меню до генерации умного списка продуктов. 

## 🛠 Стек технологий
* **Язык:** Kotlin
* **UI-фреймворк:** Jetpack Compose (декларативный подход, очень похож на React)
* **Система сборки:** Gradle (Kotlin DSL)

## 🔄 Шпаргалка для React-разработчиков
Если вы пришли из веба или React Native, вот прямые аналоги концепций, чтобы вы быстрее освоились:

| В мире React / Web | В мире Android / Compose | Описание |
| :--- | :--- | :--- |
| `package.json` | `app/build.gradle.kts` | Файл конфигурации модуля. Здесь в блоке `dependencies { ... }` мы подключаем сторонние библиотеки. |
| `npm install` | Кнопка **Sync Now** | После добавления зависимости в `build.gradle.kts` нужно нажать `Sync Now` (Синхронизировать) вверху экрана, чтобы скачались пакеты. |
| `index.js` / `App.js` | `MainActivity.kt` | Главная точка входа. Отсюда (из блока `setContent`) начинается отрисовка всего интерфейса. |
| JSX Компоненты | `@Composable` функции | Вместо функций, возвращающих HTML-теги, мы пишем функции с аннотацией `@Composable`, внутри которых вызываем другие UI-элементы (например, `Text()`, `Button()`). |
| `Props` | Аргументы функции | Данные передаются в `@Composable` функции точно так же — через обычные аргументы. |
| `useState()` | `remember { mutableStateOf() }` | Локальное состояние компонента. При его изменении Compose автоматически перерисовывает (recompose) интерфейс. |
| `useEffect()` | `LaunchedEffect()` | Место для сайд-эффектов (например, первичная загрузка данных с сервера при открытии экрана). |
| `className` / CSS / Стили | `Modifier` | Почти каждый UI-компонент принимает параметр `modifier`. Через него задаются отступы, размеры, скругления: `Modifier.padding(16.dp).fillMaxWidth()`. |
| `console.log()` | `Log.d("Tag", "Текст")` | Вывод логов для дебага. Искать их нужно не в консоли терминала, а в специальной вкладке **Logcat** (внизу окна студии или VS Code). |

## 📂 Архитектура рабочего модуля (`app/`)
Основная разработка ведется внутри директории `app/`. Вот за что отвечает каждая её часть:

* **`app/src/main/java/com/example/smartmeal/`**
  Здесь лежит весь исходный код:
  * `ui/theme/` — глобальные настройки дизайна. В файле `Color.kt` мы фиксируем нашу "съедобную" палитру (аппетитный оранжевый, свежий зеленый, чистый белый). В `Type.kt` настраивается типографика, чтобы сохранять в интерфейсе "много воздуха".

* **`app/src/main/res/`**
  Папка для статических ресурсов:
  * `values/strings.xml` — **важно:** мы не хардкодим текст прямиком в элементы. Все строковые значения ("Сгенерировать меню", "Корзина") выносятся сюда в виде переменных.
  * `drawable/` и `mipmap/` — директории для иконок и графики.

* **`app/src/main/AndroidManifest.xml`**
  Главный конфигурационный файл Android. Здесь мы запрашиваем системные разрешения (например, доступ к интернету `INTERNET` для связи с API).

## ⚙️ Инфраструктура сборки (Корень `client/SmartMeal/`)
Чтобы у всей команды была идентичная среда без конфликтов версий, мы используем **Gradle Wrapper**:
* **`gradlew` (Mac/Linux) / `gradlew.bat` (Windows)** — скрипты, которые сами скачивают нужную версию сборщика. Вам не нужно устанавливать Gradle вручную.
* **`gradle/libs.versions.toml`** — единый каталог, где хранятся версии всех библиотек для централизованного управления.

## 🚀 Как начать работу
1. Откройте директорию `client/SmartMeal/` в VS Code (или Android Studio).
2. При использовании VS Code убедитесь, что установлены официальные расширения для Kotlin.
3. Дождитесь успешной синхронизации системы Gradle.
4. Вы можете запускать приложение на подключенном смартфоне или эмуляторе (кнопка Run).
