# smartmeal-android
# 🥗 Умный ужин (SmartMeal)

Мобильное приложение, которое избавляет от ежедневного вопроса «Что приготовить?». Оно составляет персонализированное меню, подбирает рецепты и автоматически генерирует умный список продуктов для похода в супермаркет или заказа онлайн.

## 🎯 Концепция и Дизайн
* **Стиль:** Чисто, аппетитно, минимализм. Много воздуха.
* **Цветовая палитра:** «Съедобные» цвета — зеленый (свежесть), оранжевый (аппетит), белый.
* **Главное правило:** От открытия приложения до готового списка покупок — менее 5 кликов.

## 🛠 Технологический стек
Проект построен на современной клиент-серверной архитектуре:

* **Frontend (Мобильное приложение):**
  * Язык: Kotlin.
  * UI-фреймворк: Jetpack Compose.
  * Архитектура: Single Activity, Material 3.
* **Backend (Сервер и API):**
  * Язык: Python 3.12+
  * Фреймворк: Django & Django REST Framework.
* **База данных:**
  * СУБД: PostgreSQL.

## 📂 Структура проекта (VS Code)

Репозиторий разделен на две независимые части:

```text
SmartMeal/
├── client/                 # Нативное Android-приложение
│   ├── app/                # Основной модуль приложения (зависимости в build.gradle.kts)
│   │   └── src/main/java/  # Исходный код Kotlin (UI, логика)
│   ├── gradle/             # Настройки системы сборки Wrapper
│   └── gradlew             # Скрипт для запуска сборки без установки Gradle
│
└── server/                 # Серверная часть на Python
    ├── accounts/           # Приложение аккаунтов пользователей (модель юзеров)
    ├── config/             # Главные настройки Django (settings.py, urls.py)
    ├── meals/              # Приложение логики (модели рецептов, ингредиентов)
    ├── sql/                # SQL-скрипт для создания основных таблиц базы данных
    ├── manage.py           # Главный скрипт управления сервером
    ├── requirements.txt    # Список Python-зависимостей для команды
    └── .env.example        # Пример файла .env с переменными окружения
```

## 🚀 Как запустить проект локально

### Запуск Backend (Django)
Откройте терминал и перейдите в папку сервера: `cd server`

#### Установка зависимостей:
1. Создайте виртуальное окружение: 
   * Windows: `python -m venv venv`
   * Mac/Linux: `python3 -m venv venv`
2. Активируйте окружение:
   * Windows: `venv\Scripts\activate`
   * Mac/Linux: `source venv/bin/activate`
3. Установите зависимости: `pip install -r requirements.txt`

#### Задать переменные окружения:
1. Скопировать файл `cp .env.exampe .env`
2. Указать значения переменных окружения

#### База данных:
1. Существующая в PostgreSQL или создать вручную новую
2. Указать имя бд, пароль и пользователя в PostgreSQL, порт (если отличается от дефолтного) в файле .env

#### Генерация secret key:
1. Запустить консоль `python manage.py shell`
2. В консоли:
```bash
from django.core.management.utils import get_random_secret_key
get_random_secret_key()
``` 
3. Скопировать сгенерированный ключ и вставить в .env SECRET_KEY

#### Старт сервера:
1. Сделайте миграции `python manage.py migrate`
2. Запустите локальный сервер: `python manage.py runserver`
3. Сервер будет доступен по адресу: `http://127.0.0.1:8000/`

### Запуск Frontend (Android)
1. Откройте папку `client/SmartMeal` в VS Code (или Android Studio).
2. Дождитесь автоматической синхронизации Gradle (или нажмите Sync).
3. Запустите проект на подключенном физическом устройстве или эмуляторе.

## 👥 Командная работа
* Не коммитим папку `venv/`, `__pycache__/`, локальную БД `db.sqlite3` и секретные ключи (`.env`).
* Файлы миграций базы данных (`server/meals/migrations/`, `server/accounts/migrations`) **обязательно** добавляем в коммиты!
* Ветки создаем под каждую новую фичу (например: `feature/menu-generator`, `fix/login-screen`).
